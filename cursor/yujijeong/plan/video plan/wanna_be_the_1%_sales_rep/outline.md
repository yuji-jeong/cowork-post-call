## Section 1 — The Mindset Shift (3:00 – 5:00)

This is the frame that makes everything else click.

"Before we go into details, I wanna briefly talk about something 99% are not doing and why they can't be that 1% sales rep."

**The problem:** Most people use Claude or ChatGPT like a vending machine. They open a chat, explain their whole context, get an output, copy paste and close it. Next call — same thing. They're re-briefing every single day. This is where you waste a lot of time, repeating over and over, getting inconsistent results full of mistakes, and you end up fixing it yourself. so why even use AI at all right?

**The shift:** You need to start treating Claude as part of your team — a personal admin assistant you've already onboarded, not a seasonal temp suffering dementia. To do this, you have to build "skills" but you do it once. Claude then knows your process. It knows your business. It knows what details matter in your CRM. And it runs the same way every time without you explaining anything.

Chat = a temp you re-brief every single day.
Skills = an admin assistant you've already onboarded.

**Line to use on camera:** "Once you understand this shift, you no longer have to jump between all these tools to get what you need. Claude becomes the one place where all of it happens. You show up, type one command, and it's done."

*On screen: animation — HubSpot, Gmail, your transcription tool, all mapped out from Claude as a hub. Then cut to the old flow: Firefly tab → HubSpot tab → Gmail tab, manual copy-paste between each. Then the new flow: one Claude command → all three update automatically.*

**What changes with skills:** You're no longer summarising calls, no longer copying and pasting, no longer writing follow-up emails from scratch. The repetitive part — that stops you from getting more leads — that's what the skill handles.

**What a skill actually is — say this before moving on:**

"What is a skill? A skill is just a very organised prompt. Prompt is when you type something into the ChatGPT like 'hey what disease is this'? It's exactly that. Except instead of typing out long instructions every time, you write it once in a plain text file, step by step. And Claude follows that exact process every time you run it. No code. No technical setup. Just clear instructions in plain English. And I've already written this for you so you can copy it for free."

*On screen: open the skill `.md` file in a text editor — show it's plain readable text, not a config file or code.*

**Bridge — say this immediately after showing the file:**

"And when you type `/post-call` in Claude — this file is exactly what it runs. That slash command triggers this exact set of instructions. That's the whole thing."

*On screen: show the slash command being typed, then cut back to the `.md` file — draw the connection visually.*

## What NOT to Do (13:30 – 14:00)

**Don't approve every change blindly.**

The skill asks before it moves a deal stage. That confirmation step is there for a reason. If the transcript was cut off, if there was an off-the-record conversation after the call ended, or if the deal context is more nuanced than what was discussed — Claude doesn't know that. You do.

One read of the reasoning before you hit confirm. That's all it takes.

*On screen: example where Claude flags a deal stage move that the rep decides to hold off on — they decline, the rest of the workflow runs without it.*
